## What to build

Integrate **Recordatorios** into the Ticket **finish** flow: when a user finishes a Ticket that has `client_id` and service lines, show a block listing each distinct Service on the Ticket with checkbox, interval controls (presets + Personalizado), and editable last-service date (default `ticket_date`). Lines with an existing schedule for that Client+Service are **checked by default**; new pairs **unchecked**. Only checked lines call upsert on confirm. Tickets without `client_id` show Spanish explanation that scheduling requires a linked Client. Duplicate service lines on one Ticket still yield one schedule per pair. Requires `tickets.write`.

## Acceptance criteria

- [ ] Finish flow shows Recordatorios when applicable; skipped when no `client_id` with clear message
- [ ] Existing schedule lines checked by default; new pairs unchecked; unchecked lines do not write DB
- [ ] Checked lines upsert schedule with correct `last_service_at` and computed `next_due_at`
- [ ] Finish + schedule save error handling does not leave ambiguous finished state (document behavior in PR comment if needed)
- [ ] Browser: finish a Ticket with linked Client and service; verify schedule row updates on management page
